# Security policy / Политика безопасности

Do not disclose credentials or an exploitable vulnerability in a public issue. Contact the repository owner privately through the verified contact channel listed on their GitHub profile. Include affected version, reproduction steps and impact; omit real user data.

Не публикуй ключи или рабочую уязвимость в открытом Issue. Свяжись с владельцем репозитория через подтверждённый контакт в его GitHub-профиле. Укажи версию, шаги воспроизведения и влияние, но не прикладывай реальные данные пользователей.

Supported security work includes SSRF prevention in URL imports, short-lived media tokens, secret isolation, dependency updates and safe handling of provider redirects.
